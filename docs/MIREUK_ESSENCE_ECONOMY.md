| 단위 | 정수(精髓) |

### 1-2. 세계관적 의미

미력(味力)이 일반적으로는 요리의 신비한 힘이라면, **미력의 정수**는 식란이 극도로 격렬한 순간 또는 매우 순수한 정화 의식에서만 결정화되는 농축된 에너지 결정체다.

> "식란 한 번에 불꽃이 피어나지만, 정수는 천 번의 불꽃이 하나로 모일 때 생겨난다."
> — 행상인 포코

일반 영업에서는 절대 얻을 수 없다. 평범한 조리와 서빙으로 축적되는 주방코인, 또는 전투와 영업의 결과물인 골드와 달리, 미력의 정수는 **특수한 조건**에서만 세상에 나타난다. 이 때문에 유랑 미력사들이 이 정수를 구하기 위해 각지를 떠돌다 미미의 식당에 발길을 멈추게 된다.

### 1-3. 역할 분리 원칙

| 화폐 | 획득 루트 | 소비처 | 희귀성 |
|------|----------|-------|-------|
| 골드 | 영업 서빙 → 영구 누적 | 행상인(도구 구매/업그레이드) | 낮음 (매 스테이지) |
| 주방코인 | 스테이지 클리어 별점 보상 | 상점(레시피/테이블/인테리어/직원) | 중간 (진행마다) |
| **미력의 정수** | 특수 NPC 서빙 / 보스 정화 / 스토리 보상 | **유랑 미력사 고용/강화** | 높음 (비정기적) |

세 화폐의 소비처가 서로 겹치지 않아야 한다. 미력의 정수는 유랑 미력사 전용 화폐로, 골드나 주방코인으로는 대체 불가다.

## 3. 특수 손님 NPC — 미력 나그네 (味力の旅人)

### 3-1. 기본 정의

| 항목 | 내용 |
|------|------|
| type_id | `mireuk_traveler` |
| 한국어 명칭 | 미력 나그네 |
| 아이콘/표기 | 💠 (기본 이모지 폴백) / 전용 픽셀 아이콘 생성 예정 (Phase 48-2 AD) |
| 시각적 특징 | 낡은 도롱이를 걸친 여행자 실루엣. 손에 발광하는 작은 결정체(미력의 정수)를 쥐고 있다. HUD 위에 보랏빛 입자 파티클 오라. |
| 외형 크기 | 기존 손님 이모지와 동일 크기, 색상은 보라-금 배합 |

### 3-2. 등장 조건 및 확률

| 항목 | 내용 |
|------|------|
| 잠금 해제 조건 | 7-1 클리어 이후 (그룹2 진입) |
| 기본 등장 확률 | 영업 세션 시작 시 16% 확률로 해당 세션에 1회 등장 예약 |
| 등장 타이밍 | 세션 시작 후 60~90초 사이 무작위 시점에 홀 진입 |
| 챕터별 등장 확률 보정 | 7~15장: 16% / 16~24장: 20% / 엔드리스: 12% (도전성 유지) |
| 세션당 최대 등장 횟수 | 1회 (중복 등장 없음) |

### 3-3. 행동 패턴

| 항목 | 수치 | 근거 |
|------|------|------|
| patience_mult | 1.5 | 여유로운 여행자. 급하지 않음. |
| reward_mult (골드) | 0.8 | 여행자는 돈이 없다. 골드 보상은 낮다. |
| reward_mult (미력의 정수) | — | 서빙 점수 기반 드롭 (아래 로직 참조) |
| 요청 레시피 | 상위 30% 등급 레시피 중 무작위 | 미식가적 취향 |
| 자리 선호 | 창가 테이블 우선 (격자상 가장 외곽 열) | 연출용, 강제는 아님 |

### 3-4. 서빙 완료 시 미력의 정수 드롭 로직

```
서빙 점수 = (남은 인내심 비율) × (레시피 등급 보정)
  - 남은 인내심 80% 이상 → 3 정수 드롭
  - 남은 인내심 40~79%   → 2 정수 드롭
  - 남은 인내심 39% 이하  → 1 정수 드롭
  - 서빙 실패 (기다리다 퇴장) → 0 정수

레시피 등급 보정:
  - 전설/고급 레시피 서빙 → 보정 없음 (기본 드롭)
  - (선택) 나그네가 요청한 레시피를 정확히 서빙 시 보너스 +1 정수
```

드롭된 정수는 VFXManager로 플로팅 텍스트 "💠 +N 정수!" 연출. 기존 골드 플로팅과 색상 구분 (보라색 텍스트).

### 3-5. 대화 훅 (DialogueManager 연결)

| 트리거 시점 | 대화 ID (예시) | 내용 요약 |

## 4. 소비처 테이블 — 유랑 미력사 고용/강화 비용

### 4-1. 등급 체계 개요

유랑 미력사는 4개 등급(초급/중급/고급/전설)으로 분류된다. 등급이 높을수록 영업 특화 스킬이 강력하다. 세부 스킬 설계는 Phase 48-2~3 범위.

### 4-2. 고용/강화 비용 테이블

| 등급 | 고용 비용 | 재고용 비용 | 강화 비용 (1→2단계) | 강화 비용 (2→3단계) | 해고 시 환급 |
|------|---------|-----------|-----------------|-----------------|-----------|
| 초급 | 4 정수 | 2 정수 | 3 정수 | 6 정수 | 없음 |
| 중급 | 12 정수 | 6 정수 | 10 정수 | 20 정수 | 없음 |
| 고급 | 35 정수 | 18 정수 | 30 정수 | 60 정수 | 없음 |
| 전설 | 80 정수 | 40 정수 | 70 정수 | 140 정수 | 없음 |

**재고용**: 한 번 고용했다가 해고한 미력사를 다시 고용할 때. 처음 만남의 "인연 비용"이 없으므로 할인 적용.

**환급 없음**: 정수는 희귀 화폐이므로 해고 시 돌려받지 못한다. 고용 신중하게 결정하도록 유도.

### 4-3. 동시 고용 가능 인원 (상한선 설계 예시)

| 조건 | 동시 고용 상한 |
|------|-------------|
| 기본 | 1명 |
| 7~12장 클리어 | 2명 |
| 13~18장 클리어 | 3명 |
| 19~24장 클리어 | 4명 |

상한을 두어 "정수를 한꺼번에 소진하는" 패턴을 방지하고, 콘텐츠 진행에 따른 점진적 확장감을 제공한다.

## 5. 경제 밸런스 설계

### 5-1. 목표 획득 속도

| 시나리오 | 목표 시간 |
|---------|--------|
| 신규 플레이어가 초급 미력사 1명을 처음 고용 | 약 3~4시간 실제 플레이 |
| 중급 미력사 1명 추가 고용 | 초급 이후 약 2~3시간 추가 |
| 고급 미력사 첫 고용 | 중급 이후 약 5~7시간 추가 |

**근거**: 그룹2 진입 후 미력 나그네 세션당 평균 0.32회 등장(확률 16% × 2 예상치) × 드롭 평균 2 정수 = 세션당 기댓값 약 0.64 정수. 1세션 약 3분. 3시간 = 60세션 × 0.64 = 약 38 정수. 보스 보상 + 스토리 이정표 포함 시 초급(4정수) 고용은 40분~1시간 내 가능. 중급(12정수)은 약 3시간. 이 속도는 Phase 48-2 구현 후 플레이 테스트를 통해 조정.

### 5-2. 최대 보유량 캡

최대 보유량 상한을 **999 정수**로 설정. 실질적으로는 무제한과 동일. 상한이 존재한다는 사실 자체로 "귀한 화폐" 느낌을 주는 한편, 일반 플레이어가 상한에 부딪힐 일은 없다.

### 5-3. 시즌/이벤트 추가 획득 여지

| 이벤트 유형 | 설명 |
|------------|------|
| 미력 폭풍의 밤 (주간 이벤트) | 특정 요일 엔드리스 플레이 시 나그네 등장 확률 2배 |
| 식란 기념일 (연간 이벤트) | 스테이지 클리어 보상에 정수 소량 추가 |
| 달인 의뢰 (일일 퀘스트) | "이 레시피 5번 서빙" → 1~2 정수 보상 |

이벤트 설계는 Phase 48 이후 별도 기획 페이즈에서 구체화. 이 문서는 기본 경제 프레임만 확정.

### 5-4. 피로도 방지 원칙

1. **농사(grinding) 방지**: 미력의 정수를 얻기 위해 같은 스테이지를 반복하면 효율이 낮도록 설계한다. 보스 재클리어 보상이 없는 이유.
2. **부재 시에도 보존**: 정수에는 시간 소멸(decay)이 없다. 오래 쉬었다 돌아온 플레이어가 모아둔 정수를 잃지 않는다.
3. **명확한 진행 지표**: HUD에 현재 보유량 + "다음 고용까지 N 정수 더 필요" 안내를 선택적으로 표시(Phase 48-2 UI 설계 반영).
4. **보상의 가시성**: 미력 나그네 서빙 시 VFX + 사운드 연출로 "특별한 것을 얻었다"는 확실한 피드백 제공. 조용히 숫자만 올라가선 안 된다.

```

  data.mireukEssence = 0         // 보유 정수 수량
  data.mireukEssenceTotal = 0    // 누적 획득 정수 (업적용)

  data.mireukEssence = 0
```
