## 0. 본 문서의 위치

기존 [SERVICE_SCENE_REDESIGN.md](SERVICE_SCENE_REDESIGN.md)는 Phase 50~52에서 진행한 **아이소메트릭 격자 + 3레이어 분리 렌더링** 방향이다. Phase 76 손님 NPC 다양성 확장을 진행하면서 **신규 92×92 스프라이트가 기존 48×64 규격과 호환되지 않고**, 의자 정합 문제도 근본적으로 해결되지 않은 점이 드러나, 영업씬 자체를 카이로 소프트(*Cafeteria Nipponica*, *大盛り食堂物語* 등) 스타일의 사선뷰 식당 시뮬레이션 구조로 **재설계**하기로 결정했다.

본 문서는 새 방향의 **지향점·핵심 원칙·기준 시안**을 정리한다. 구체적 페이즈 분할, 마이그레이션 계획, 코드 변경 명세는 별도 스펙 문서에서 다룬다.

## 3. 핵심 원칙

### 3-1. 좌석은 "세트" 단위로 관리한다

- 테이블·의자를 별도 오브젝트로 두지 않는다.
- `table_4seat.png`처럼 **테이블 1개 + 의자 N개를 하나의 합성 스프라이트로 처리**한다.
- 합성 세트는 자기가 보유한 좌석 슬롯의 픽셀 좌표를 데이터로 들고 있다.

```javascript
// 예시 데이터 구조
const TABLE_SET_4SEAT = {
  texture: 'table_4seat',
  width: 64, height: 64,
  seats: [
    { id: 'n1', dx: 14, dy: 6,  dir: 'north', zOffset: -2 },
    { id: 'n2', dx: 78, dy: 6,  dir: 'north', zOffset: -2 },
    { id: 's1', dx: 14, dy: 70, dir: 'south', zOffset: +2 },
    { id: 's2', dx: 78, dy: 70, dir: 'south', zOffset: +2 },
  ],
};
```

### 3-2. 손님은 "앉기 전용" 컷을 사용한다

- 다리·하반신을 그리지 않은 **상반신 컷**(약 24×28 ~ 36×42)을 손님별로 1장씩 준비한다.
- 의자 좌표에 단순 박힘. 정합 계산 불필요.
- 걷기/대기 스프라이트와는 별도 시트.

| 상태 | 스프라이트 |
|------|----------|
| 입장·줄서기·이동 | `customer_{type}_walking.png` (전신, 기존 자산 활용 가능) |
| 착석 | `customer_{type}_seated.png` (상반신 컷, 신규 필수) |
| 식사 (선택) | `customer_{type}_eating.png` (상반신 + 음식 표현) |
| 만족 (선택) | `customer_{type}_happy.png` |

### 3-3. Y축 깊이 정렬로 가림을 만든다

```
위쪽 의자 손님 (z: BASE+1)
    ↓ 테이블이 손님 일부를 가림 (의도적 가림)
테이블 합성 세트 (z: BASE+2)
    ↓ 손님이 테이블 위에 보임 (자연스러운 안착)
아래쪽 의자 손님 (z: BASE+3)
```

- 의자 등받이는 테이블 합성 세트에 포함됨 → 자동으로 위쪽 손님의 뒤를 받쳐줌.
- 아래쪽 손님은 어차피 상반신 컷이라 다리가 없음 → 테이블 앞에 박혀도 어색하지 않음.

### 3-4. 사선뷰 격자 바닥

- 32×32 정도의 단일 타일을 반복 렌더링한다.
- 타일은 정탑다운이 아닌 **사선 격자 패턴**(가벼운 입체감)을 가진다.
- 챕터별 바닥 테마는 [기존 SERVICE_SCENE_REDESIGN.md §2-2](SERVICE_SCENE_REDESIGN.md) 매핑을 그대로 계승한다.

### 3-5. 공간 분할 (360×640 기준 잠정안)

```
┌─────────────────────────────────────────┐
│ HUD (타이머 / 골드 / 챕터)         │ 32px
├──────────────┬──────────────────────────┤
│              │                          │
│  주방 카운터 │   다이닝홀 (격자 바닥)   │
│  + 셰프     │                          │
│              │   - 좌석 세트 ×3~4       │
│  (좌측 1/3)  │   - 동시 8~12명          │
│              │                          │ 528px
│              │                          │
│              │      ┌──────────┐        │
│              │      │ 입구·대기열 │     │
│              │      └──────────┘        │
├──────────────┴──────────────────────────┤
│ 컨트롤 바 (요리 카드 / 일시정지 등)   │ 80px
└─────────────────────────────────────────┘
                  360px
```

- 좌측 1/3 주방 / 우측 2/3 다이닝홀은 V9 시험에서 검증된 비율.
- 입구·대기열은 다이닝홀 우측 상단에 배치.
- 좌석 세트 수와 배치는 챕터별 가변(작은 식당 → 큰 식당으로 진화).

## 4. 폐기 또는 보류되는 기존 방향

| 항목 | 처리 | 이유 |
|------|------|------|
| 아이소메트릭 다이아몬드 격자 | **폐기** | 카이로식 사선뷰 격자로 대체. 새 시야가 더 손님 시인성·가구 정합에 유리. |
| 테이블 앞면/뒷면 분리 (`_back`/`_front`) | **폐기** | 사전 합성 좌석 세트로 통합. depth 계산 단순화. |
| 손님 독립 스프라이트 + Y depth+50 | **부분 계승** | 분리 렌더링 컨셉은 유지하되, 좌석 세트 + 상반신 컷 조합으로 단순화. |
| 신규 92×92 손님 풀바디 | **보류** | 카이로식 상반신 컷으로 재발주 필요. 기존 자산은 재활용 또는 폐기 검토. |
| `table_lv{n}_occupied.png` 컴포짓 | **폐기 확정** | 좌석 세트 + 손님 단순 박힘으로 완전 대체. |
