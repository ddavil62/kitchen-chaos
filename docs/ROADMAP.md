## 게임 재미 구조 리워크 (Phase 58+)

### 배경

Phase 57까지 **콘텐츠 물량**(스테이지 143 / 레시피 284 / 적 57 / 셰프 15)은 충분히 확보됨.
그러나 재미 설계 분석 결과 아래 3가지 구조적 한계가 드러남:

| 항목 | 현황 | 문제 |
|------|------|------|
| 난이도 곡선 | HP↑·시간↓ 선형 증가 | 임계점/스파이크 없음 → 긴장감 평탄 |
| 도구·셰프 시너지 | 독립 작동 | 조합 보너스 0 → "아하 모먼트" 부재 |
| 행상인 선택 무게 | 수치 환전소 | 선택이 다음 판을 바꾸지 않음 |

**리워크 목표**: 스테이지마다 "상태 변화"가 체감되는 구조로 전환. 쇼츠/클립 소재 자동 발생.

### Phase 58 — 행상인 로그라이크 분기 선택 (full, both) ✅ 완료 (2026-04-22)

**목표**: 행상인을 "수치 환전소"에서 "게임 방향이 갈리는 분기점"으로 전환.

**핵심 메커닉**: 매 행상인 방문 시 기존 도구 구매 옵션에 더해 **"3택 1 분기 카드"**를 제시. 선택은 되돌릴 수 없음.

### Phase 59 — 영업 페이즈 러시아워 시스템 (full, ui)

> 참고 문서(예정): `docs/SERVICE_RUSH_HOUR_DESIGN.md`

**목표**: 영업 페이즈를 "시간 깎기" time attack에서 "긴장감 스파이크가 있는 웨이브 구조"로 전환.

**핵심 메커닉**: 기존 영업 페이즈를 **3구간(초반 / 러시아워 / 마무리)**으로 분할.

**구간별 변화**:

| 구간 | 지속 | 손님 밀도 | 특수 요소 |
|------|------|----------|----------|
| 초반 | 40% | 기존 수준 | 일반 주문만 |
| **러시아워** | 30% | **동시 3명 입장** | VIP / 미식가 / 급한 손님 등장 빈도 ↑, 특수 주문(풀코스, 복합 주문) |
| 마무리 | 30% | 기존 수준 | 러시 후 여유 구간, 팁 보상 |

**러시아워 극복 보상**:
- 골드 ×1.5 버프
- 명성(reputation) +N — 명성 시스템 신규 도입 검토
- 미력의 정수 확정 드롭

**실패 시 페널티**:
- 러시 구간에서 손님 이탈 시 기존보다 강한 평판 하락
- "망할 뻔 → 극복" 클립 구조 확보

**구현 범위**:
- ServiceScene: 러시 구간 트리거 + 비주얼 연출(배경 명도↑, BGM 템포↑)
- CustomerSpawner: 구간별 인내심/동시 입장 수 재설계
- 특수 주문 데이터 (`specialOrderData.js` 신규) — 풀코스/복합/시간제한 주문 정의
- UI: 러시아워 시작 "RUSH HOUR!" 배너 + 남은 시간 게이지
- 영업 페이즈 밸런스 재조정 (143 스테이지 전부는 아니고 그룹별 대표값)

**의존성**: Phase 58 완료 후 권장 (행상인에서 "러시 완화" 분기 카드 등장 가능)
